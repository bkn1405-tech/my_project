2. Справочная Информация по Командам Git/

  2.1 НАЧАЛО: НАСТРОЙКА И СВЯЗЬ С СЕРВЕРОМ
  git remote add origin git@github.com:bkn1405-tech/my_project.git

3. Первая Отправка В Основную Ветку
  git push -u origin master - отправка первого тестового файла на GitHub в ветку master
4. Проверка Изменении и Статуса
   4.1. git status  - проверили, есть ли измененные или новые файлы (красные/зеленные)
   4.2. git diff - поросмотр точных изменении в коде (зеленные строки со знаком +)
   4.3. git log --online - вывод кароткую историю все сделанных коммитов.
   4.4. git log -p просмотр подробной истории изменения кода во всех комитах.

5. РАБОТА С НОВЫМИ ВЕТКАМИ (ТЕКУЩАЯ ЗАДАЧА)
   5.1. git checkout -b work-diary(название ветки в нашем примере) создали ветку work-diary
   5.2. touch steps.txt или nano steps.txt - создали файл для записи шагов по работе.
   5.3. git add steps.txt - подгатовка файла к сохранению.
   5.4. git commit -m "Начал вести дневник работы, записал 1-шаг" - зафиксировал коммит дневника
   5.5. git -u origin work-diary - отправил новую ветку и файл на GitHub
